NimBull GPT Proxy Helper

This document explains how to use the /proxy endpoint to interact with the Bullhorn API via the NimBull GPT proxy.

üß† Supported Operations

Use the type query parameter to choose the Bullhorn operation mode:

TypeDescription
queryFiltered entity queries using where
searchFull-text search using query
entityFetch a single record by ID
metaGet metadata for a Bullhorn entity

üí° Example Calls

Get enabled skills

GET /proxy?type=query&entity=Skill

Defaults to where=enabled=true and fields=id,name,enabled

Search for candidates by keyword

GET /proxy?type=search&entity=Candidate&query=developer

Get a specific job order

GET /proxy?type=entity&entity=JobOrder&id=12345

Get metadata for an entity

GET /proxy?type=meta&entity=Candidate

üîÑ Caching

  • meta results are cached in-memory for 15 minutes per entity.
  • This reduces redundant API calls and improves performance.
